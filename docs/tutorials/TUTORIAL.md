## Prerequisites

### Software Requirements

- Python 3.9 or higher
- Strainphase installed (`pip install strainphase[full]`)
- PacBio HiFi sequencing data (BAM files)
- Variant calls (VCF files from Clair3 or similar)

### Input Data Requirements

**BAM files:**
- Sorted and indexed (`.bam` + `.bam.bai`)
- Aligned to reference genome (minimap2 recommended)
- PacBio HiFi reads (CCS reads with ≥99% accuracy)

**VCF files:**
- Variant calls from Clair3, DeepVariant, or GATK
- Compressed and indexed (`.vcf.gz` + `.vcf.gz.tbi`)
- Contains SNPs with allele frequency (AF) and depth (DP) information

**Reference genome:**
- FASTA format
- Indexed (`.fasta.fai` generated by `samtools faidx`)
- Contig naming format: `MAGNAME_contig_N` (for longitudinal analysis)

## Data Preparation

### Step 1: Generate Reference Index

```bash
# Index reference FASTA
samtools faidx combined_bins.fasta

# Verify index was created
ls -l combined_bins.fasta.fai
```

### Step 2: Prepare BAM Files

```bash
# For each sample, align reads and sort
minimap2 -ax map-hifi -t 16 combined_bins.fasta sample_T1.fastq.gz \
  | samtools sort -@ 8 -o sample_T1.sorted.bam

# Index BAM
samtools index sample_T1.sorted.bam

# Verify BAM quality
samtools stats sample_T1.sorted.bam | grep "average quality"
```

### Step 3: Generate VCF Files

```bash
# Using Clair3 (recommended for PacBio HiFi)
run_clair3.sh \
  --bam_fn=sample_T1.sorted.bam \
  --ref_fn=combined_bins.fasta \
  --threads=16 \
  --platform=hifi \
  --model_path=/path/to/clair3_models/hifi \
  --output=variants/sample_T1 \
  --include_all_ctgs

# Index VCF
tabix -p vcf variants/sample_T1/pileup.vcf.gz
```

### Step 4: Optional Contig Filtering

Create a file listing high-quality contigs:

```bash
# Filter contigs by coverage and breadth
samtools depth sample_T1.sorted.bam | \
  awk '{sum[$1]+=$3; count[$1]++} END {
    for (contig in sum) {
      if (count[contig] > 1000 && sum[contig]/count[contig] > 20) {
        print contig
      }
    }
  }' > high_quality_contigs.txt

# Or use a TSV with metadata
cat > contig_filter.tsv <<EOF
contig	coverage	breadth
MAG_01_contig_1	45.2	0.98
MAG_01_contig_2	38.7	0.95
MAG_02_contig_1	52.1	0.99
EOF
```

## Single Contig Analysis

### Basic Usage

Process a single contig from one sample:

```bash
strainphase run \
  --bam sample_T1.sorted.bam \
  --vcf variants/sample_T1/pileup.vcf.gz \
  --contig MAG_01_contig_1 \
  --length 150000 \
  --sample T1 \
  --output haplotypes_MAG_01_contig_1.tsv
```

### With Custom Parameters

Adjust parameters for your data:

```bash
strainphase run \
  --bam sample_T1.sorted.bam \
  --vcf variants/sample_T1/pileup.vcf.gz \
  --contig MAG_01_contig_1 \
  --length 150000 \
  --sample T1 \
  --output haplotypes.tsv \
  --window-size 5000 \
  --max-reads 500 \
  --min-mapq 30 \
  --max-mismatch 0.015 \
  --seed 42 \
  --log-level DEBUG
```

**Parameter explanations:**
- `--window-size`: Analysis window size in bp (larger = more SNVs, slower)
- `--max-reads`: Subsample to this many reads per window (reduces memory)
- `--min-mapq`: Minimum mapping quality (higher = stricter filtering)
- `--max-mismatch`: Max Hamming distance for graph edges (lower = stricter)
- `--seed`: Random seed for reproducibility

### Output Format

The output TSV contains:

| Column | Description |
|--------|-------------|
| `contig` | Contig ID |
| `sample` | Sample ID |
| `track_id` | Linked haplotype track identifier |
| `span_start` | Track start position (1-based) |
| `span_end` | Track end position (1-based, exclusive) |
| `span_bp` | Track length in bp |
| `n_windows` | Number of windows spanned |
| `n_snvs` | Number of SNVs in consensus |
| `mean_weight` | Mean abundance (mixture proportion) |
| `total_supporting_reads` | Total high-confidence reads |
| `mean_confidence` | Mean assignment confidence |
| `consensus` | SNV profile (pos:base pairs) |

**Example:**
```tsv
contig	sample	track_id	span_start	span_end	span_bp	n_windows	n_snvs	mean_weight	total_supporting_reads	mean_confidence	consensus
MAG_01_contig_1	T1	T0001	1000	12000	11000	7	45	0.65	123	0.92	1245:A|2340:G|3450:T|...
MAG_01_contig_1	T1	T0002	1000	8000	7000	5	28	0.35	68	0.88	1245:G|2340:G|3450:C|...
```

## Longitudinal Multi-Sample Analysis

### Full Pipeline

Analyze multiple timepoints to track strain dynamics:

```bash
strainphase longitudinal \
  --samples T1,T2,T3,T4,T5 \
  --bams mapping/{sample}.sorted.bam \
  --vcfs variants/{sample}/pileup.vcf.gz \
  --reference combined_bins.fasta \
  --output-dir results/longitudinal \
  --mags MAG_01,MAG_02,MAG_03 \
  --contig-filter high_quality_contigs.txt \
  --window-size 3000 \
  --max-reads 300 \
  --min-anchor-weight 0.15 \
  --rescued-min-weight 0.02 \
  --log-level INFO
```

**Parameter explanations:**
- `--samples`: Comma-separated sample IDs (no spaces)
- `--bams`: Path template with `{sample}` placeholder
- `--vcfs`: Path template with `{sample}` placeholder
- `--mags`: Process only these MAGs (optional, default: all)
- `--contig-filter`: File with allowed contigs (optional)
- `--min-anchor-weight`: Min abundance to serve as anchor (0-1)
- `--rescued-min-weight`: Min abundance after rescue (0-1)

### Rescue Mechanism

Low-abundance strains in one timepoint can be "rescued" if they:
1. Match a high-abundance strain (anchor) in another timepoint
2. Consensus distance ≤ `rescue_match_distance` (default 0.01)
3. Share ≥ `min_shared_for_rescue` SNVs (default 3)

This is especially powerful for detecting transient blooms or rare persistent strains.

### Output Files

```
results/longitudinal/
├── lineages.tsv                    # Main output: lineage table
├── longitudinal_summary.tsv        # Summary statistics
├── T1.rescued.tsv                  # Per-sample haplotypes (post-rescue)
├── T2.rescued.tsv
├── T3.rescued.tsv
├── T4.rescued.tsv
└── T5.rescued.tsv
```

**lineages.tsv format:**

| Column | Description |
|--------|-------------|
| `lineage_id` | Unique lineage identifier |
| `mag` | MAG name |
| `contig` | Contig ID |
| `sample` | Sample/timepoint ID |
| `track_id` | Track ID within sample |
| `span_start` | First SNV position in consensus |
| `span_end` | Last SNV position in consensus |
| `span_bp` | span_end − span_start (bp between first and last SNV) |
| `n_windows` | Windows in this track |
| `total_span` | Lineage span in bp (last SNV − first SNV across all samples) |
| `abundance` | Mean EM mixture proportion (weight) |
| `reads` | Reads assigned to this track |
| `total_reads` | Total non-junk reads in windows |
| `n_snvs` | SNVs in consensus |
| `consensus` | SNV profile |
| `n_timepoints` | How many timepoints have this lineage |

## Interpreting Results

### Identifying Strain Dynamics

**Persistent strains:**
```bash
# Find lineages present in ≥3 timepoints
awk -F'\t' '$NF >= 3' results/longitudinal/lineages.tsv | \
  cut -f1 | sort -u
```

**Selective sweeps:**
```bash
# Track abundance changes over time for lineage L000042
grep "L000042" results/longitudinal/lineages.tsv | \
  sort -t$'\t' -k4,4 | \
  awk -F'\t' '{print $4, $10}'  # sample, mean_weight
```

**Transient blooms:**
```bash
# Find lineages with high abundance in only 1 timepoint
awk -F'\t' '$NF == 1 && $10 > 0.3' results/longitudinal/lineages.tsv
```

### Consensus Comparison

Compare consensus sequences between lineages:

```python
import pandas as pd

# Load lineages
df = pd.read_csv('results/longitudinal/lineages.tsv', sep='\t')

# Parse consensus
def parse_consensus(cons_str):
    return dict(x.split(':') for x in cons_str.split('|'))

# Get lineage consensus
lineage_A = df[df['lineage_id'] == 'L000042'].iloc[0]['consensus']
lineage_B = df[df['lineage_id'] == 'L000058'].iloc[0]['consensus']

cons_A = parse_consensus(lineage_A)
cons_B = parse_consensus(lineage_B)

# Find differences
shared_pos = set(cons_A.keys()) & set(cons_B.keys())
diffs = [(pos, cons_A[pos], cons_B[pos])
         for pos in shared_pos if cons_A[pos] != cons_B[pos]]

print(f"Shared positions: {len(shared_pos)}")
print(f"Differences: {len(diffs)}")
print(f"Similarity: {1 - len(diffs)/len(shared_pos):.3f}")
```

### Visualization

Create abundance plots over time:

```python
import pandas as pd
import matplotlib.pyplot as plt

df = pd.read_csv('results/longitudinal/lineages.tsv', sep='\t')

# Filter to one contig
contig_df = df[df['contig'] == 'MAG_01_contig_1']

# Pivot for plotting
pivot = contig_df.pivot_table(
    index='sample',
    columns='lineage_id',
    values='mean_weight',
    fill_value=0
)

# Plot
pivot.plot(kind='area', stacked=True, figsize=(12, 6))
plt.xlabel('Timepoint')
plt.ylabel('Relative Abundance')
plt.title('Strain Dynamics: MAG_01_contig_1')
plt.legend(title='Lineage', bbox_to_anchor=(1.05, 1), loc='upper left')
plt.tight_layout()
plt.savefig('strain_dynamics.png', dpi=300)
```

## Parameter Optimization

### Quick Parameter Sweep

Test different parameter combinations:

```bash
strainphase sweep --quick --output-dir sweep_results
```

This tests a reduced parameter grid (~2-5 minutes).

### Full Parameter Sweep

```bash
strainphase sweep --output-dir full_sweep_results
```

Tests 324 configurations across 4 scenarios:
- `simple_2hap`: 2 haplotypes, 3 timepoints
- `sweep_2hap`: 2 haplotypes with selective sweep
- `complex_4hap`: 4 haplotypes, 5 timepoints
- `low_abundance`: 3 haplotypes with rare strain

### Interpreting Sweep Results

```python
import json

with open('sweep_results/sweep_summary.json') as f:
    summary = json.load(f)

# Find best configuration for your scenario
best = max(summary['scenario_results']['complex_4hap'],
           key=lambda x: x['metrics']['f1_score'])

print(f"Best F1 score: {best['metrics']['f1_score']:.3f}")
print(f"Parameters: {best['config']}")
```

### Recommended Parameters by Scenario

**High coverage (>50×), simple community:**
```bash
--window-size 5000 \
--max-mismatch 0.015 \
--merge-distance-threshold 0.005
```

**Low coverage (<30×), complex community:**
```bash
--window-size 3000 \
--max-mismatch 0.02 \
--merge-distance-threshold 0.01 \
--min-weight-for-anchor 0.10
```

**Longitudinal with rare strains:**
```bash
--min-anchor-weight 0.15 \
--rescued-min-weight 0.01 \
--min-shared-for-rescue 5
```

## Troubleshooting

### No haplotypes found

**Possible causes:**
1. Insufficient SNV density (< 3 SNVs per window)
2. Very low coverage (< 10× mean depth)
3. Window size too large
4. VCF filtering too aggressive

**Solutions:**
```bash
# Check SNV density
bcftools query -f '%CHROM\t%POS\n' variants.vcf.gz | \
  awk '{print $1}' | sort | uniq -c

# Reduce window size
--window-size 2000

# Lower AF filter
# (modify config.af_range in Python API)
```

### Too many haplotypes

**Possible causes:**
1. Sequencing errors not filtered
2. max_mismatch_frac too high
3. Repeat regions causing false clusters

**Solutions:**
```bash
# Increase MAPQ threshold
--min-mapq 30

# Reduce mismatch tolerance
--max-mismatch 0.01

# Filter VCF to high-quality variants
bcftools view -i 'QUAL>30 && DP>20' input.vcf.gz -O z -o filtered.vcf.gz
```

### Memory issues

**Solutions:**
```bash
# Reduce max reads per window
--max-reads 200

# Process one MAG at a time
--mags MAG_01

# Process one contig at a time (single-contig mode)
strainphase run --contig MAG_01_contig_1 ...
```

### Slow performance

**Solutions:**
```bash
# Use smaller windows
--window-size 2000

# Subsample reads more aggressively
--max-reads 100

# Process MAGs in parallel (separate jobs)
for mag in MAG_01 MAG_02 MAG_03; do
  strainphase longitudinal --mags $mag ... &
done
wait
```

## Advanced Usage

### Python API

```python
from strainphase import HaplotyperConfig, process_contig, results_to_dataframe

# Custom configuration
config = HaplotyperConfig(
    window_size=3000,
    max_mismatch_frac=0.02,
    min_weight_for_anchor=0.15,
    rescue_match_distance=0.01,
    random_seed=42,
)

# Process contig
results = process_contig(
    bam_path="sample.sorted.bam",
    vcf_path="variants/pileup.vcf.gz",
    contig_id="MAG_01_contig_1",
    contig_length=150000,
    config=config,
    sample_id="T1",
)

# Convert to DataFrame
import pandas as pd
records = results_to_dataframe({"MAG_01_contig_1": results})
df = pd.DataFrame(records)

# Analyze
print(df[['track_id', 'mean_weight', 'n_snvs']])
```

### Batch Processing

Process multiple samples in parallel:

```bash
#!/bin/bash
samples=(T1 T2 T3 T4 T5)
mags=(MAG_01 MAG_02 MAG_03)

# Process each MAG in parallel
for mag in "${mags[@]}"; do
  (
    echo "Processing $mag..."
    strainphase longitudinal \
      --samples $(IFS=,; echo "${samples[*]}") \
      --bams mapping/{sample}.sorted.bam \
      --vcfs variants/{sample}/pileup.vcf.gz \
      --reference combined_bins.fasta \
      --output-dir results/$mag \
      --mags $mag \
      --log-level WARNING
  ) &
done
wait

echo "All MAGs processed"
```
